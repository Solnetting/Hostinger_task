Product Design — Hostinger Challenge

Fixing hPanel's notification overload

hPanel's "Your to-dos" surfaces 53 nudge types in a single dismissable list. Renewal reminders sit next to promotional offers. Security alerts share visual weight with upsell prompts. Nothing feels urgent because everything does.

This is the design process for replacing that list with a tiered, contextual notification system — and the structural finding that surfaced along the way.

Role Product Designer Type Design Challenge Deliverable System + interactive prototype
The problem

Everything looks urgent, so nothing is

hPanel's home screen has a persistent "Your to-dos" list that collects every pending action across the account: domain renewals, email verification, SSL notices, promotional offers, security tips, upsell prompts. All rendered with the same card format, the same visual weight, the same dismiss button.

The result is a list users learn to ignore. A domain renewal due in 3 days competes visually with "Add a phone number to your account." Neither feels important. CTA conversion is low despite the list being always visible.

The audit found 53 nudge types. All of them defaulted to the home screen. None had priority logic. The list treated urgency as a content problem rather than a design problem.

The core question wasn't how to make the list shorter. It was: what is each nudge actually asking the user to do, and does the surface match the stakes?

Analysis

Mapping urgency against type

Each nudge was mapped along two axes: urgency — what happens if ignored, and how quickly — and scope — product-specific or account-level. That mapping revealed four natural clusters defined by behavior, not feature area.

Routing criteria

Each of the 53 nudge types was put through three questions:

QuestionT1T2T3T4
What happens if ignored? Service disruption or billing failure — immediate Real loss (expiry, lockout) within days Suboptimal outcome, recoverable Nothing — purely optional
Is there a deadline? Yes — hours to days Yes — days to weeks Open-ended or no urgency No deadline
Is there a natural product-page context? No — affects entire account Yes, but home is also appropriate Yes — belongs exclusively on the relevant page N/A — it's a promotion, not a task

T1 = immediate harm, account-wide. Dual-tier routing = a deadline plus a product-page home — same nudge as a T2 home card and a T3 product-page indicator simultaneously.

The system

A 4-tier triage model

Each tier maps to a specific surface, a dismissal behavior, and a display cap. The goal: the surface itself communicates urgency before a user reads a single word.

TierLabelSurfaceDismissalCap
T1CriticalPersistent strip, top of content — always visibleNeverNone
T2Time-sensitiveHome screen nudge cardsSnooze (reappears when <3 days left)2 items max
T3ContextualInline on the relevant product pageResolves on action — no close, no snoozeUnlimited
T4OffersOffers drawer (pull-only, topbar pill)Read = dismissedUnlimited

T2 caps at 2. Overflow is a passive chip — never a "View all." T4 carries no badge on its pill: a badge is push in disguise.

Design decisions

Five behaviors that define the system

T1 follows the user everywhere

The T1 strip sits above both sidebar and content at full viewport width — not scoped to the content area. A cancelled hosting plan isn't a home-page problem; it's system-level. The strip persists on Domains, Emails, VPS — every page — until resolved.

T2 is capped at 2. No "View all."

When a card is resolved or snoozed, the next queued item slots in. Overflow shows as a passive chip: "+ 2 more need attention" — gray, non-interactive. No "View all" link, no separate destination. A separate page would only be reachable during overflow, never from primary nav — the original problem with a better label.

T3 never appears on home. The sidebar dot is the bridge.

T3 nudges live exclusively on their product page. The sidebar tab gets a red dot — no number. A dot says something here; a badge says manage this list. T3 clears only when the underlying issue is resolved. Snoozing the T2 version hides the home card but keeps the dot — snooze is a home action, not a global resolution.

Dual-tier routing: same nudge, two surfaces

Some nudges qualify as both T2 and T3 simultaneously. Resolving from either surface clears both. Tier describes where a nudge surfaces, not what it is.

T1 stays crisp when T4 opens

When the Offers drawer opens, a dim overlay covers the content area — but not T1. The overlay starts below the banner. A user with a critical billing issue can still act on it while browsing deals.

T4 is pull-only — "Offers" is the right word

No badge on the pill. The label "Promotions & offers" was shortened to "Offers" — honest about what it is. The user opens it when they choose to. Never before.

Hi-fi screens

Nine states, one system

Screen 0 · Healthy home

Zero nudges. Home defaults to website cards and quick actions. Nothing competes for attention when there's nothing to act on.

Screen · T1 alone

Single critical issue. Full-width strip, non-dismissable. Content sits normally below it — the strip doesn't displace the page.

Screen · Multi-T1

Two simultaneous critical issues collapse into one strip with a numbered summary. No stacking, no double-height banner.

Screen · T1 + T2

Critical issue and two time-sensitive items. Two urgency levels, two separate surfaces — neither competes with the other.

Screen · T2 only

No T1 active. "Needs attention" leads the page and disappears when both items are resolved or snoozed.

Screen · T2 overflow

Cap at 2. The passive chip (+ 2 more need attention) signals the queue without creating a destination. Non-interactive by design.

Screen · T3 contextual (Domains)

T1 persists on product pages. The domain-transfer nudge surfaces inline on Domains, where the user manages it. The sidebar dot led them here — T3 never pushed.

Screen · T4 offers drawer

Pull-only. No badge on the Offers pill. The dim overlay starts below T1, which stays crisp. The user chose to open this.

Screen · T4 shelf alt

Below-fold deals shelf — an alternative to the drawer. Passive, ambient, no pill required. Present when the user scrolls; invisible if they don't.

Structural finding

There is no Account tab — and that's the root cause

While placing nudges into tiers, a category emerged that didn't fit any product page: email verification, phone, account-level 2FA. hPanel's sidebar has no Account section. These nudges had no sidebar home, so they defaulted to home — not because that was right, but because there was nowhere else.

The triage process surfaced IA debt, not just notification clutter. The bloated list was a symptom of a structural gap: the product had grown account-level features without a dedicated account surface. Notifications aren't objects — they're routing policies. The list was a symptom of having no routing logic at all.

The topbar avatar becomes a fifth surface — a lightweight, always-present account anchor. Account nudges live in the dropdown: scoped, unobtrusive, contextually correct. Whether the avatar scales as hPanel grows is an open question — one the triage process surfaced, not resolved.

Interactive prototype

Try the system

All four tiers are live. Navigate the sidebar to see T3 contextual nudges on Domains, Emails, and Dark web monitoring. Open Promotions & offers (no badge — pull-only). Click the avatar for the account nudge. Snooze or resolve T2 cards on Home.
