Product Design — Hostinger Challenge

Fixing hPanel's notification overload

hPanel's "Your to-dos" surfaces 53 nudge types in one list — renewal reminders next to promo offers, security alerts sharing visual weight with upsell prompts. Nothing feels urgent because everything does. This is the system that replaced it.

Role Product Designer Type Design Challenge Deliverable System + interactive prototype
The problem

Everything looks urgent, so nothing is

53 nudge types, all rendered with the same card format and dismiss button. A domain expiring in 3 days competes visually with "Add a phone number." The question wasn't how to shorten the list — it was whether each nudge's surface matched its stakes.

Design decisions

The decisions that define the system

T1 follows the user everywhere

Full-width, above sidebar and content. Not home-scoped — a cancelled plan is a system problem, not a page problem.

T2 caps at 2 decision groups. No "View all."

Same-family nudges collapse into one card. Overflow is a passive gray chip — no destination page, because a destination only reachable during overflow is the original problem with better styling. Cards offer Snooze (auto-resurfaces near deadline), not permanent dismiss.

Setup / Continue is a protected T2 slot, not a new tier

14 setup nudges collapse into one "Finish setting up (N)" card — 1 T2 slot. Dismissable (abandoning is valid). Earns home-screen presence because setup completion is conversion-critical; doesn't compete with expiry deadlines.

No Notification Center

A Notification Center combines urgency routing with record-keeping — which recreates the 53-item list with better icons. Record-keeping belongs under Billing as an activity log, not a triage surface.

T3 never appears on home. The sidebar dot is the only bridge.

Dot = something here. Badge = manage this list. Snoozing the T2 version hides the home card but keeps the dot — snooze is a home action, not a global resolution.

Dual-tier routing: same nudge, two surfaces

Some nudges qualify as both T2 (home card) and T3 (product page indicator) simultaneously. Resolving from either clears both.

T4 is pull-only. No badge.

Drawer + passive below-fold shelf. Renewal discounts ride the T2 card as a chip — they don't earn their own T4 slot. The shelf gives PMs an impression channel without a single push notification.

The "Refer" topbar button becomes the Offers pill

hPanel's current topbar has a permanent "Refer a friend" button next to search and the account icon. A referral is not a utility action — placing it there overstates its priority. It's replaced by the Offers pill: a pull-only entry to the T4 drawer, no badge, never pushed. The referral is still accessible. The topbar gets back to doing one job.

Access requests are T3, not T2

The requested_access family has no deadline — a person is waiting for a decision, not a clock ticking. T2's 2-slot cap protects deadline items; an indefinitely-pending access request can displace a domain expiring in 7 days. These route T3: contextual to the specific resource being accessed, actioned in place, sidebar dot signals the waiting request from home. Closeable — only the account owner bears the consequence of ignoring.

Open questions

  • Avatar at scale. Works for 1–2 account nudges. At 5–6 it becomes a dense dropdown.
  • Snooze lifecycle. What happens after N snoozes on a perpetually-in-grace-period domain?
  • T3 dot discoverability. A red sidebar dot assumes the user knows to navigate there.
  • T1 false-positive recovery. If a payment clears but the banner doesn't, support volume spikes.
Hi-fi screens

Nine states, one system

Screen 0 · Healthy home

Zero nudges. Home defaults to website cards and quick actions. Nothing competes for attention when there's nothing to act on.

Screen · T1 alone

Single critical issue. Full-width strip, non-dismissable. Content sits normally below it — the strip doesn't displace the page.

Screen · Multi-T1

Two simultaneous critical issues collapse into one strip with a numbered summary. No stacking, no double-height banner.

Screen · T1 + T2

Critical issue and two time-sensitive items. Two urgency levels, two separate surfaces — neither competes with the other.

Screen · T2 only

No T1 active. "Needs attention" leads the page and disappears when both items are resolved or snoozed.

Screen · T2 overflow

Cap at 2. The passive chip (+ 2 more need attention) signals the queue without creating a destination. Non-interactive by design.

Screen · T3 contextual (Domains)

T1 persists on product pages. The domain-transfer nudge surfaces inline on Domains, where the user manages it. The sidebar dot led them here — T3 never pushed.

Screen · T4 offers drawer

Pull-only. No badge on the Offers pill. The dim overlay starts below T1, which stays crisp. The user chose to open this.

Screen · T4 shelf alt

Below-fold deals shelf — an alternative to the drawer. Passive, ambient, no pill required. Present when the user scrolls; invisible if they don't.
