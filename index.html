Product Design — Hostinger Challenge

Fixing hPanel's notification overload

hPanel's "Your to-dos" surfaces 53 nudge types in a single dismissable list. Renewal reminders sit next to promotional offers. Security alerts share visual weight with upsell prompts. Nothing feels urgent because everything does.

This is the design process for replacing that list with a tiered, contextual notification system — and the structural finding that surfaced along the way.

Role Product Designer Type Design Challenge Deliverable System + interactive prototype
The problem

Everything looks urgent, so nothing is

hPanel's home screen has a persistent "Your to-dos" list that collects every pending action across the account: domain renewals, email verification, SSL notices, promotional offers, security tips, upsell prompts. All rendered with the same card format, the same visual weight, the same dismiss button.

The result is a list users learn to ignore. A domain renewal due in 3 days competes visually with "Add a phone number to your account." Neither feels important. CTA conversion is low despite the list being always visible.

The audit found 53 nudge types. All of them defaulted to the home screen. None had priority logic. The list treated urgency as a content problem rather than a design problem.

The core question wasn't how to make the list shorter. It was: what is each nudge actually asking the user to do, and does the surface match the stakes?

Analysis

Mapping urgency against type

Each nudge was mapped along two axes: urgency — what happens if the user ignores this, and how quickly — and scope — is this tied to a specific product (Domains, Emails, Security) or to the account itself?

That mapping revealed four natural clusters. Not by feature area, but by behavior: how large the time window is, whether real harm follows inaction, and whether there's a more contextually appropriate surface outside the home screen.

A domain expiring in two days is categorically different from a promotional offer. A payment failure is not the same as a suggestion to add 2FA. Once those differences are explicit, the right surfaces follow naturally.

The system

A 4-tier triage model

Each tier maps to a specific surface, a dismissal behavior, and a display cap. The goal: the surface itself communicates urgency before a user reads a single word.

Tier Label Surface Dismissal Cap
T1 Critical Persistent strip, top of content — always visible Never None
T2 Time-sensitive Home screen cards Snooze (reappears when <3 days left) 2 items max
T3 Contextual Inline on the relevant product page Dismiss on page — gone permanently Unlimited
T4 Offers & tips Promotions drawer (pull-only, topbar pill) Read = dismissed Unlimited

T2 is hard-capped at 2 cards because showing more recreates the original problem. If a third item qualifies as time-sensitive, the oldest card gets demoted or resolved before the new one appears.

T4 carries no badge on its topbar pill. A badge is push in disguise — it manufactures urgency where none exists. T4 is pull-only by definition. The user opens it when they want to.

Design decisions

T1 stays crisp when the T4 drawer opens

When a user opens the Promotions drawer, a dim overlay covers the content area. But the T1 banner — which sits between the topbar and content — cannot be suppressed. A user with a critical billing issue should still be able to act on it even with the drawer open.

The dim overlay is scoped to start below the T1 banner. T1 has no dismiss button by design: non-dismissable is the point. Open the Promotions drawer in the prototype below to see this behavior in action.

Structural finding

There is no Account tab — and that's the root cause

While placing nudges into tiers, a category emerged that didn't fit any product page: nudges that belong to the user's account, not to any hosted service. Email verification. Phone number. Account-level 2FA.

hPanel's sidebar is organized by product: Websites, Domains, Emails, Security, VPS, Help. There is no Account section. These nudges had no sidebar home, so they defaulted to the home screen — not because that was the right place, but because there was nowhere else.

The triage process surfaced IA debt, not just notification clutter. The bloated list was a symptom of a structural gap: the product had grown account-level features without a dedicated account surface.

The fix treats the topbar avatar as a fifth surface — a lightweight, always-present account anchor. Account nudges live in the avatar dropdown: scoped, unobtrusive, and contextually correct. No new sidebar tab required.

The avatar has a badge in the prototype below — it represents a pending email verification nudge. Click it to see how the account nudge surfaces without interrupting the product flow.
