Product Design — Hostinger Challenge

Fixing hPanel's notification overload

hPanel's "Your to-dos" surfaces 53 nudge types in a single dismissable list. Renewal reminders sit next to promotional offers. Security alerts share visual weight with upsell prompts. Nothing feels urgent because everything does.

This is the design process for replacing that list with a tiered, contextual notification system — and the structural finding that surfaced along the way.

Role Product Designer Type Design Challenge Deliverable System + interactive prototype
The problem

Everything looks urgent, so nothing is

hPanel's home screen has a persistent "Your to-dos" list that collects every pending action across the account: domain renewals, email verification, SSL notices, promotional offers, security tips, upsell prompts. All rendered with the same card format, the same visual weight, the same dismiss button.

The result is a list users learn to ignore. A domain renewal due in 3 days competes visually with "Add a phone number to your account." Neither feels important. CTA conversion is low despite the list being always visible.

The audit found 53 nudge types. All of them defaulted to the home screen. None had priority logic. The list treated urgency as a content problem rather than a design problem.

The core question wasn't how to make the list shorter. It was: what is each nudge actually asking the user to do, and does the surface match the stakes?

Analysis

Mapping urgency against type

Each nudge was mapped along two axes: urgency — what happens if the user ignores this, and how quickly — and scope — is this tied to a specific product (Domains, Emails, VPS, Dark web monitoring) or to the account itself?

That mapping revealed four natural clusters. Not by feature area, but by behavior: how large the time window is, whether real harm follows inaction, and whether there's a more contextually appropriate surface outside the home screen.

A domain expiring in two days is categorically different from a promotional offer. A payment failure is not the same as a suggestion to add 2FA. Once those differences are explicit, the right surfaces follow naturally.

Routing criteria

Each of the 53 nudge types was put through three questions:

QuestionT1T2T3T4
What happens if ignored? Service disruption or billing failure — immediate Real loss (expiry, lockout) within days Suboptimal outcome, recoverable Nothing — purely optional
Is there a deadline? Yes — hours to days Yes — days to weeks Open-ended or no urgency No deadline
Is there a natural product-page context? No — affects entire account Yes, but home is also appropriate Yes — belongs exclusively on the relevant page N/A — it's a promotion, not a task

If a nudge scored "immediate harm, account-wide, no product page," it's T1. If it had a deadline but a natural home on a product page, it was eligible for dual-tier routing — surfacing as both T2 (home card) and T3 (product page indicator) simultaneously. The routing is additive, not exclusive.

The system

A 4-tier triage model

Each tier maps to a specific surface, a dismissal behavior, and a display cap. The goal: the surface itself communicates urgency before a user reads a single word.

TierLabelSurfaceDismissalCap
T1CriticalPersistent strip, top of content — always visibleNeverNone
T2Time-sensitiveHome screen nudge cardsSnooze (reappears when <3 days left)2 items max
T3ContextualInline on the relevant product pageDismiss on page — gone permanentlyUnlimited
T4OffersOffers drawer (pull-only, topbar pill)Read = dismissedUnlimited

T2 is hard-capped at 2 cards. When a card is resolved or snoozed, the next queued item auto-slots in. Overflow shows as a passive count chip — not a "View all" link. There is no separate notification page for T2.

T4 carries no badge on its topbar pill. A badge is push in disguise — it manufactures urgency where none exists. T4 is pull-only by definition. The user opens it when they want to.

Design decisions

Five behaviors that define the system

T1 follows the user everywhere

The T1 strip sits above both the sidebar and the content frame at full viewport width — a direct child of the screen, not scoped to the content area. A cancelled hosting plan isn't a home-page problem; it's a system-level one. The strip is present on Domains, Emails, VPS — every page the user navigates to — until resolved. Scoping T1 to home would imply the problem goes away when the user navigates. It doesn't.

T2 is capped at 2. No "View all."

The cap is the feature. When a T2 card is resolved or snoozed, the next queued item automatically slots in. Overflow is shown as a passive count chip: "+ 2 more need attention" — gray, no arrow, non-interactive. There is no "View all" link and no separate notification page for T2. A separate destination would be an IA orphan: only reachable during overflow, never from primary nav. Reachable means discoverable — and that discoverability would need a permanent entry point. Which is the original problem all over again.

T3 never appears on home. The sidebar dot is the bridge.

T3 nudges live exclusively on their product page. The sidebar tab gets a red dot — no number. A dot says something here. A badge says manage this list. When the user sees the Domains tab has a dot, they navigate intentionally; T3 never pushes. Snoozeing a T2 version of the same nudge hides the home card but keeps the dot — snooze is a home action, not a global resolution.

Dual-tier routing: same nudge, two surfaces

Some nudges qualify as both T2 and T3 simultaneously. "rocketman.xyz expires in 2 days" is a home card with Snooze + Renew CTA (T2) and an inline row indicator on the Domains page (T3) — same event, two surfaces, different treatment. Resolving from either surface clears both. This isn't redundancy: the Domains page is where you manage the domain, so the nudge belongs there too. Tier describes where a nudge surfaces, not what it is.

T1 stays crisp when T4 opens

When the Offers drawer opens, a dim overlay covers the content area — but not the T1 banner. The overlay is scoped to start below T1. A user with a critical billing issue can still act on it while browsing deals. T1 has no dismiss button. Non-dismissable is the point. Open the Offers drawer in the prototype below to see this behavior in action.

Pulling T2 into the T4 drawer would recreate the notification center with a better icon. The original problem wasn't the visual design of the list — it was routing everything to the same surface. Aggregating tiers dissolves the signal. Each tier must stay on its own surface.

T4 is pull-only — and "Offers" is the right word

The drawer carries no badge on its topbar pill. A badge is push in disguise: it manufactures urgency where none exists. T4 is user-initiated by definition. The label "Promotions & offers" was replaced with "Offers" — shorter, less commercial-feeling, and honest about what it is. The user opens it when they want to. Never before.

Structural finding

There is no Account tab — and that's the root cause

While placing nudges into tiers, a category emerged that didn't fit any product page: nudges that belong to the user's account, not to any hosted service. Email verification. Phone number. Account-level 2FA.

hPanel's sidebar is organized by product: Websites, Domains, Horizons, Emails, Reach, VPS, Dark web monitoring, Billing, Marketplace. There is no Account section. These nudges had no sidebar home, so they defaulted to the home screen — not because that was the right place, but because there was nowhere else.

The triage process surfaced IA debt, not just notification clutter. The bloated list was a symptom of a structural gap: the product had grown account-level features without a dedicated account surface. Notifications aren't objects — they're routing policies. The list was a symptom of having no routing logic at all.

The fix treats the topbar avatar as a fifth surface — a lightweight, always-present account anchor. Account nudges live in the avatar dropdown: scoped, unobtrusive, and contextually correct. No new sidebar tab required.

The avatar has a badge in the prototype below. Click it to see how the account nudge surfaces without interrupting the product flow.
